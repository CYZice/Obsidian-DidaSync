# Dida Sync 同步插件使用说明

## 插件简介
滴答清单同步插件是一款专为 Obsidian 设计的任务管理插件，可以将您的滴答清单任务同步到 Obsidian 中，或编辑 Obsidian 原生任务同步到滴答清单，实现双向同步管理。支持任务创建、编辑、删除、完成状态同步等功能。

## 前置要求
1. Content Protection 最新版
2. 拥有滴答清单账号
3. 完成 OAuth 认证（详见下文 OAuth 配置、验证教程）

## 使用方法

### 打开任务清单

**方法一：点击状态栏**
点击 Obsidian 底部状态栏的 "滴答清单" 文字。

**方法二：使用命令**
快捷键 `Ctrl/Cmd + P` 打开命令面板 → 输入 "打开滴答任务清单"。

### 在笔记中创建 Dida 任务

**方法一（原生任务同步）**
创建方式：Obsidian 自带的任务语法（`- [ ]` 格式），或者“鼠标右键 → 段落设置 → 任务列表”。

*(Pasted image 20251105202349.png)*

请按照以下顺序操作：
1. **输入任务标题**：输入任务标题时不会触发同步。
    - 关闭菜单时才触发同步。
    - 可多次手动修改任务标题并同步到滴答清单。
2. **点击“🔗 同步到滴答”**：将任务链接到 Dida Sync 插件并同步到滴答清单。
    - 在后面自动添加 `🔗Dida` 链接。
    - 点击 `🔗Dida` 链接可打开右侧任务详情。
    - 不可多次添加或修改。
3. **点击“📅 到期日期”**：设置任务的到期日期。
    - 可多次弹出菜单重新更新到期日期。
    - 可直接手动修改到期日期并同步到滴答清单。

> **注意**：
> 1. 修改任务标题、到期日期后，按照此线路双向同步：原生任务 ⇋ Dida Sync 插件 ⇋ 滴答清单
> 2. 关闭操作菜单时，自动同步（这是触发同步的条件）

**方法二（可用于 Dida 任务与 Obsidian 笔记关联）**
1. 在 Markdown 文档中，快速输入两个 `@@` 符号（注意快速输入，否则单个 `@` 符号会触发 AI 续写对话框），自动弹出任务选择建议窗口。
2. 在输入框可搜索并选择已有任务，将笔记链接到已有的任务，形成双向链接。
3. 在输入框输入新的任务标题，Enter 键可以创建新的任务，新的任务将自动添加到 Dida 任务清单列表，并且形成双向链接。

### 主任务视图（项目清单视图）
打开任务清单后，可以在右侧边栏主任务视图（项目）编辑任务。

1. **任务标签页**：方框对勾图标若不显示，请通过上面两种方法打开任务清单。
2. **任务列表**：
    - 点击任务标题、检查项计数、子任务计数，可以打开对应的任务详情标签，分母代表总数，分子代表已完成数量。
    - 🗑️：用来删除任务（注意：删除任务将不再保持数据）。
    - ☁️✅：绿色对勾云朵图标，代表已经完成了同步。
    - ☁️❗：红色感叹云朵图标，代表 Obsidian 本地已经创建了任务，但是没有同步（需要连接网络进行同步）。
3. **任务详情标签**：任务标题与内容、检查项、子任务 3 个标签。
4. **保存**：记得编辑任务后保存（否则任务不会同步）。
5. **项目清单**：点击项目清单，可以折叠/展开，前后两个数字代表“主任务+子任务”数量。
6. **任务添加按钮**：在该项目清单中添加任务。
7. **同步按钮**：用来手动同步任务。
8. **到期日期**：点击设置到期日期（可设置循环任务）。

## 插件命令
插件提供以下命令（通过 `Ctrl/Cmd + P` 调用），如需要，可以自行设置快捷键：

- **手动双向同步**：将 Obsidian 任务同步到滴答清单，同时获取滴答清单任务。
- **在项目中添加任务**：可选择选择项目并设定日期创建任务（用于任何窗口下快速创建任务）。
- **打开滴答任务清单**：首次使用插件可通过此命令打开任务清单。

## 插件设置
在插件设置中，你可以设置定期清理已完任务或者重置任务数据。

- **自动清理设置**：自动清理已完任务，默认情况是关闭的，为了防止数据堆积卡顿可自行设置清理间隔时间。
- **启用原生任务同步**：关闭后，原生任务（`- [ ]`）将不再同步到滴答清单。
- **数据重置**：当 Obsidian 任务数据移除（无显示、数据破坏等），可重置数据。

### 任务视图设置
- **默认视图模式**：可设置为时间段视图、列表视图。
- **每小时高度**：时间段视图的左侧时间刻度可自定义设置。
- **时间段视图起始时间**：可设置起始时间（例如：避开凌晨时间）。

## 故障排除

### 认证失败
1. 检查 Client ID 和 Client Secret 是否正确。
2. 确认重定向 URI 配置正确。
3. 检查网络连接。
4. 所有方法都试过了，请等待一会儿，重启 Obsidian。
5. 若仍然失败，请重新按照之前的步骤，重新创建应用再认证。

### 同步失败
1. 检查网络连接。
2. 确认滴答清单账号正常。
3. 尝试重新认证。

## 注意事项
1. 原生任务创建、编辑时，会实时同步到滴答清单（注意：若同步异常，请重新编辑任务）。
2. 由于滴答清单 API 接口 (v1) 不支持获取已完任务，所以目前 Dida Sync 不打算开放 "显示已完任务功能"。需要编辑已完任务时，请在移动端滴答清单 app 进行操作。
3. 将任务移动到其他项目清单功能尚未开发，请在移动端滴答清单 app 中进行操作。
4. 创建项目清单功能尚未开发，请在移动端滴答清单 app 中进行操作。
5. 滴答清单中的任务未完成，但是已经归档，任务数据仍然会同步到 Obsidian，当归档的未完任务数量比较多时，将直接影响同步的速度，以及加载的速度，因此不建议归档未完成的任务。
