### 📅 Dida Sync (滴答清单同步) 开源重构架构文档

#### 1. 核心应用架构与工作流 (Overview)
该插件旨在将 **Obsidian 本地 Markdown 任务** 与 **滴答清单 (Dida365 / TickTick)** 进⾏深度双向同步。
当前 main.js 采用整体挂载的方式，主要依赖以下运行机制：
*   **状态存储**: 任务列表全量通过 `plugin.settings.tasks` 存在本地 `data.json` 内进行持久化。
*   **交互界面**: 在 Obsidian 的右侧边栏提供自定义 View（任务面板）和独立的日历时间线视图（TimeBlock/Timeline）。
*   **反向解析**: 解析 Markdown 笔记里的带有 `[🔗Dida]` 的任务字符串，通过正则建立 Obsidian 文本与滴答服务端任务的关联。

#### 2. 当前核心模块（类）职责拆解 (Modules)

在乱码和合并的结构中，插件实际由以下几大功能域组成，重构时应当将其拆分到对应文件中：

**🟢 A. 入口与设置管理 (Core)**

*   `DidaSyncSettingTab`: 插件设置面板，提供 OAuth (Client ID / Secret) 绑定与本地同步周期等设置配置。

**🔵 B. 视图与UI层 (Views)**
*   `TaskView (extends ItemView)`: 滴答清单的侧边栏视图入口。包含大量的 DOM 操作（生成子任务、复选框、时间筛选器），以及重度的拖拽排序逻辑。

**🟡 C. 同步与数据管理 (Managers)**
*   `NativeTaskSyncManager`: Obsidian 原生任务同步管理器，实时监控网络状态 (`navigator.onLine`)，利用正则（`/^(\s*)-\s*\[([ x])\].../`）读取和替换当前 Obsidian MD 文档中的任务打卡状态和到期日期。
*   `RepeatTaskManager`: 管理周期性任务以及它的副本日历。
*   `RRuleParser`: 专门解析典型的循环规则字符串 (如 `"FREQ=DAILY;INTERVAL=1"`) 并计算下一次到期时间。
*   **(内部 API Fetch)**: 目前混在 Plugin 类内的各类关于 `https://dida365.com/oauth/` 及 REST API 的请求，包括创建任务、读取任务。这部分强烈建议抽象出隔离的 API Client 类。

**🟣 D. 交互弹窗组件 (Modals & UI Components)**
代码中涵盖了大量的 Obsidian Modal 和界面增强逻辑。
*   `DatePickerModal`: 日期和时间维度的选择器。
*   `RepeatSettingsModal` & `CompactRepeatSettings`: 管理“每周/每月重复一次”的循环任务界面。
*   `AddTaskModal` / `AddTaskToProjectModal`: 提供了多种形式的捷径方式直接在 Obsidian 把临时想法发送到收集箱或特定项目。
*   `TaskActionMenu` / `TaskSuggestionPopup`: 编辑器内部扩展，当打出特殊指令时显示下拉补全菜单，或对文本按右键展示上下文操作菜单。
*   `AuthUrlModal`: 仅用于帮助受限环境下手动完成 OAuth 复制粘贴的弹窗辅助。
